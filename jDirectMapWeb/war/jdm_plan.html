1 - Introduction:

 

The jQuery Direct Map or jDirectMap is a project to enable users to relate two different structural data sets. To support clear design and optimize performance for hierarchical structure, combination of spin down with tree control is used to selectively hide and show information or nodes.



The project was started as subproject of the SmooKs Direct Mapping project. It is all part of ERPEL project of the Institute of Software Technology and Interactive Systems (ISIS) a Faculty of Computer Science of the Vienna University of Technology.
The jDirectMap project will be supported and implemented under supervision of both the Electronic Commerce Group and Business Informatics Group
mainly Philipp Liegl, Christian Huemer, Marco Zapletal , as well as others.

Currently the jQuery UI Tree plug-in is in development, but not yet documented (as it's not part of a jQuery UI release). is a tree page on the UI documentation wiki page with more details about the project:

jQuery Tree project

 

As part of this project a wide research on currently available jQuery tree plug-ins was conducted. Results (full report to be published shortly) shown that there are fairly robust plugins with lots of features to consider:

DynaTree

jsTree

zTree

 

As part of the project all of these major plug-ins will be considered and implemented in demo. The plan is to implement basic versions extending each of these tree plugins and leave the selection process to the user. Certain minimal requirements and functional specification with precondition need to be defined and followed in each version.

 

 

2 - Timeline

 

Project Kick-Off was on 11.11.2011

 

The parent project SmooKs Direct Mapping required a user interface to enable user to define relationship between source and target document. The current Eclipse Plugin for SmooKs was not anymore discontinued by JBoss. The design was not able to support native direct mapping. Due to this reason there was need to create new tool. The project started with top design overview what is to achieve. A web based tool which will be cloud ready had strong influence on the decision not to create plugins into any desktop development tools. A reasearch on proper web based technology to fulfill requirements was committed.

 

Progress meeting 24.11.2011

 

As the technology a strong growing JavaScript Library jQuery - "write less do more" was agreed. The independence of the jDirectMap project will enable to deliver tool to the jQuery community as jQuery plug-in. For implementation already existing plug-ins will be used. The research of these tools was committed.

 

 

Progress meeting 22.12.2011

 

As part of this project a wide research on currently available jQuery tree plug-ins was conducted. Results (full report to be published shortly) shown that there are fairly robust plugins with lots of features to consider:

DynaTree

jsTree

zTree

 

For the jDirectMap map element a jqGrid plug-in was chosen.

 

 

Progress meeting 08.02.2012

 

First demo version was presented. The functionality and design present was approved. A discussion around design of functions was conducted. Also the limits of JQuery and technical difficulties presented. Some of the plug-ins behave unexpected in multi plug-in environment. The export of the mapping result as well as the possible import of the source and target file was outlined.
